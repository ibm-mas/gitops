Maximo Application Suite GitOps
===============================================================================

A GitOps approach to managing Maximo Application Suite using ArgoCD and Helm.

## Repository Structure

This repository is organized into several key directories that work together to deploy and manage MAS/SLS/AIService instances:

### `root-applications/`
Contains ArgoCD Application definitions that follow the **App of Apps** pattern. These are deployed to the ArgoCD management cluster and create a hierarchical structure:

- **[`ibm-mas-account-root`](root-applications/ibm-mas-account-root/)** - Top-level Application that generates Cluster Root Applications via ApplicationSet
  - **[`ibm-mas-cluster-root`](root-applications/ibm-mas-cluster-root/)** - Generates Instance Root Applications via ApplicationSet and manages cluster-level prerequisites
    - **[`ibm-mas-instance-root`](root-applications/ibm-mas-instance-root/)** - Generates MAS instance deployments and manages instance-level resources
    - **[`ibm-aiservice-instance-root`](root-applications/ibm-aiservice-instance-root/)** - Generates AI Service instances via ApplicationSet
      - **[`ibm-aiservice-tenant-root`](root-applications/ibm-aiservice-tenant-root/)** - Manages individual AI Service tenant deployments
    - **[`ibm-mas-sls-root`](root-applications/ibm-mas-sls-root/)** - Generates Suite License Service Standalone instances via ApplicationSet

### `cluster-applications/`
Helm charts deployed to target clusters as prerequisites for MAS instances. These are referenced by Applications defined in [`ibm-mas-cluster-root`](root-applications/ibm-mas-cluster-root/):

- Operator installations (DRO, NFD, GPU, etc.)
- Certificate management
- RBAC configurations
- Cluster-level utilities

### `instance-applications/`
Helm charts for MAS instance resources. These are referenced by Applications defined in [`ibm-mas-instance-root`](root-applications/ibm-mas-instance-root/):

- MAS Suite installation and configuration
- Database deployments
- Application installations (Manage, Monitor, etc.)
- Workspace configurations

### `sls-applications/`
Helm charts for Suite License Service Standalone instances, generated by the SLS ApplicationSet in [`ibm-mas-cluster-root`](root-applications/ibm-mas-cluster-root/).

### `sub-charts/`
Utility Helm charts used as dependencies across multiple parent charts. These provide reusable components and common functionality.

### `rbac/`
RBAC (Role-Based Access Control) configurations for deploying MAS in Application Admin Mode with namespace-scoped permissions.

**Note:** The Kustomize-based RBAC in this directory is **only required** when using the [pre-install repository](https://github.com/ibm-mas/pre-install) for cluster prerequisites. If you are using `cluster_admin_role=true` in a separate ArgoCD instance to deploy cluster-level resources, the [`600-application-admin-rbac`](instance-applications/600-application-admin-rbac/) Helm chart will automatically create all necessary RBAC for the `application_admin_role`. If you have one ArgoCD instance handling both cluster and namespace resources (the default scenerio), you can skip this directory entirely.

See [`rbac/README.md`](rbac/README.md) for detailed information on:

- Deployment modes (Cluster Admin vs Application Admin)
- When to use Kustomize-based RBAC management
- Generating namespace overlays for MAS instances
- Permissions included and excluded
- Deployment workflows and troubleshooting
