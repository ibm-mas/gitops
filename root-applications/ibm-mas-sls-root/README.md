## Configuration Files

The SLS Root Application is generated by the SLS ApplicationSet in the Cluster Root Application. It does not use ApplicationSets itself, but directly renders an ArgoCD Application based on values passed from the parent ApplicationSet.

The parent ApplicationSet consumes configuration files matching this pattern:

| Configuration File Pattern | Purpose | Required |
|---------------------------|---------|----------|
| `{account.id}/icn/{cluster.id}/*/ibm-sls.yaml` | Standalone SLS instance configuration | Yes |

**Note:** The path pattern `{account.id}/icn/{cluster.id}/*/` means files are organized as `{account.id}/icn/{cluster.id}/{instance.id}/ibm-sls.yaml` in the config repository.

## Application Behavior

### SLS Application

The SLS Application ([`100-ibm-sls-app.yaml`](templates/100-ibm-sls-app.yaml)) deploys a standalone Suite License Service instance:

**Target Namespace:** `mas-{ibm_customer_number}-{subscription_id}-sls`

**Source Path:** `sls-applications/100-ibm-sls`

**Sync Policy:**

| Setting | Value | Description |
|---------|-------|-------------|
| `automated.prune` | `true` (if `auto_delete=true`) | Automatically delete resources when removed from Git |
| `automated.selfHeal` | `true` | Automatically sync when cluster state drifts from Git |
| `retry.limit` | `20` | Maximum retry attempts for failed syncs |
| `syncOptions` | `CreateNamespace=true` (if `cluster_admin_role=true`) | Automatically create target namespace |
| `syncOptions` | `RespectIgnoreDifferences=true` | Honor ignoreDifferences configuration |

**Ignore Differences:**
- ServiceAccount `imagePullSecrets` (automatically managed by OpenShift)

**Application Naming:** `sls.{ibm_customer_number}.{subscription_id}`

## Example Configuration

### Minimal values.yaml (for local testing)

```yaml
account:
  id: "production"

region:
  id: "us-east-1"

cluster:
  id: "cluster-east-1"
  url: "https://api.cluster-east-1.example.com:6443"

source:
  repo_url: "https://github.com/ibm-mas/gitops"
  revision: "main"

argo:
  namespace: "openshift-gitops"
  projects:
    apps: "mas"

ibm_sls_standalone:
  ibm_customer_number: "000"
  subscription_id: "sub123"
  sls_domain: "sls.example.com"
  dns_provider: "cis"
  cis_service_name: "my-cis-service"
  cis_crn: "crn:v1:bluemix:public:internet-svcs:global:a/..."
  run_sync_hooks: true
  sls_channel: "3.x"
  ibm_entitlement_key: "<path:secret#key>"
  sls_mongo_secret_name: "sls-mongo-credentials"
  sls_mongo_username: "<path:secret#username>"
  sls_mongo_password: "<path:secret#password>"
  sls_entitlement_file: "<path:secret#file>"
  icr_cp_open: "icr.io/cpopen"
  sls_install_plan: "Automatic"
  mongo_spec:
    storage:
      class: "gp2"
      size: "20Gi"
```
